# 常见问题

## 构建问题

### NSIS 安装包下载失败

**问题**：执行 `pnpm tauri build` 时，NSIS 下载失败，提示 TLS 证书错误：

```
Error failed to verify TLS certificate: UnknownIssuer
```

**原因**：网络环境中的代理、TUN 或防火墙对 GitHub HTTPS 做了证书接管，导致 Windows 系统不信任 GitHub 返回的证书链。

**解决方案 1（推荐）**：将 `github.com` 的证书导入 Windows “受信任的发布者”

#### 1. 获取 `github.com` 的证书

可以直接用浏览器导出：

1. 用浏览器打开 `https://github.com`
2. 点击地址栏左侧的小锁图标
3. 打开证书查看窗口
4. 选择当前站点证书，点击“导出”
5. 保存为 `.cer` 文件

#### 2. 导入证书

1. 按 `Win + R`
2. 输入 `certmgr.msc`
3. 打开 “证书 - 当前用户”
4. 进入 `受信任的发布者` -> `证书`
5. 右键空白处，选择“所有任务” -> “导入”
6. 选择刚刚导出的 `github.com` 证书文件，完成导入

#### 3. 验证

```powershell
curl.exe -L "https://github.com/tauri-apps/binary-releases/releases/download/nsis-3.11/nsis-3.11.zip"
```

如果这里已经不再报 `UnknownIssuer` / `SEC_E_UNTRUSTED_ROOT`，再重新执行：

```sh
pnpm tauri build
```

**解决方案 2**：执行 NSIS 缓存修复脚本

项目里已经提供了 NSIS 缓存检查与修复脚本：

```powershell
powershell -ExecutionPolicy Bypass -File .\tools\prepare-nsis-cache.ps1
```

如果需要自动修复本地 NSIS 缓存，执行：

```powershell
powershell -ExecutionPolicy Bypass -File .\tools\prepare-nsis-cache.ps1 -Repair
```

这个脚本会检查并补齐 Tauri 2 当前需要的 NSIS 文件。

**备注**：

- exe 文件仍然可以正常使用，构建产物位于 `src-tauri/target/release/tauri-app.exe`
- MSI 安装包位于 `src-tauri/target/release/bundle/msi/`
- 如果应用内更新和 NSIS 下载同时都访问 GitHub 失败，通常是同一个证书链问题

## 更新问题

### 应用内更新检查失败，提示 GitHub 请求失败

**问题**：应用内点击“检查更新”时失败，常见报错类似：

```text
error sending request for url (https://github.com/.../releases/latest/download/latest.json)
```

或者在终端里执行：

```powershell
curl.exe -L "https://github.com/<owner>/<repo>/releases/latest/download/latest.json"
```

出现：

```text
schannel: SEC_E_UNTRUSTED_ROOT
证书链是由不受信任的颁发机构颁发的
```

**原因**：当前网络环境对 GitHub HTTPS 做了代理或证书接管，导致 Windows 系统不信任这条证书链。浏览器有时能访问，但 Tauri updater / `curl.exe` 这类走系统 TLS 校验的程序会失败。

**解决方案**：将当前 `github.com` 的证书导入 Windows “受信任的发布者”

#### 1. 获取 `github.com` 的证书

可以直接用浏览器导出：

1. 用浏览器打开 `https://github.com`
2. 点击地址栏左侧的小锁图标
3. 打开证书查看窗口
4. 选择当前站点证书，点击“导出”
5. 保存为 `.cer` 文件

#### 2. 导入证书

1. 按 `Win + R`
2. 输入 `certmgr.msc`
3. 打开 “证书 - 当前用户”
4. 进入 `受信任的发布者` -> `证书`
5. 右键空白处，选择“所有任务” -> “导入”
6. 选择刚刚导出的 `github.com` 证书文件，完成导入

#### 3. 验证

重新执行：

```powershell
curl.exe -L "https://github.com/<owner>/<repo>/releases/latest/download/latest.json"
```

如果已经不再报 `SEC_E_UNTRUSTED_ROOT`，应用内更新通常也会恢复正常。

**备注**：

- 这个问题通常与代理、TUN、抓包工具或 HTTPS 证书接管有关
- 如果你使用的是 GitHub Releases 作为更新源，这类问题更容易在本地网络环境中出现
