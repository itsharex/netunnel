**2. 配 GitHub 仓库机密**
到你的 GitHub 仓库设置里添加：

- `Secrets and variables` → `Actions` → `Secrets`
- 新增 `TAURI_SIGNING_PRIVATE_KEY`
- 新增 `TAURI_SIGNING_PRIVATE_KEY_PASSWORD`

再到：

- `Secrets and variables` → `Actions` → `Variables`
- 新增 `TAURI_UPDATER_PUBLIC_KEY`

对应关系是：

- 私钥全文 -> `TAURI_SIGNING_PRIVATE_KEY`
- 私钥密码 -> `TAURI_SIGNING_PRIVATE_KEY_PASSWORD`
- 公钥全文 -> `TAURI_UPDATER_PUBLIC_KEY`

**3. 发第一个版本**
先确认这三个版本号一致：

- [package.json](D:\git-projects\tauri-vue-template\package.json)
- [src-tauri/Cargo.toml](D:\git-projects\tauri-vue-template\src-tauri\Cargo.toml)
- [src-tauri/tauri.conf.json](D:\git-projects\tauri-vue-template\src-tauri\tauri.conf.json)

以后可以直接用：

```sh
pnpm bump 2.9.4
```

然后提交并打 tag：

```sh
git add .
git commit -m "release: v2.9.4"
git tag v2.9.4
git push
git push origin v2.9.4
```

这会触发工作流：

- [.github/workflows/release.yml](D:\git-projects\tauri-vue-template\.github\workflows\release.yml)

它会生成安装包、签名文件和 `latest.json`。

**4. 发布 Release**
GitHub Actions 现在配置的是 `releaseDraft: true`，所以它会先生成草稿 Release。你去 GitHub 上把这个 draft release 发布掉。

正式发布后，更新地址就是：

```text
https://github.com/<你的用户名>/<你的仓库>/releases/latest/download/latest.json
```

模板里已经按这个地址接好了。

**5. 测试应用内更新**
测试方式建议这样：

1. 先发布一个旧版本，比如 `v2.9.4`，安装到本机
2. 再把版本 bump 到 `v2.9.5` 并发布
3. 打开旧版本应用
4. 在设置页点“检查更新”
5. 应该能看到新版本并下载安装，安装后自动重启

**两个注意点**
- 第一次发布时，没有“更老版本”可更新，这是正常的。第一次主要是建立更新源。
- 只有通过这套签名发布出去的版本，后续应用内更新才稳定可用。
